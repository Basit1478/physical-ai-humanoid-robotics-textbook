**NEVER commit the `.env` file to git!** The `.env` file contains sensitive API keys and secrets.

## Setup Instructions

1. **Copy the example file:**
   ```bash
   cp .env.example .env
   ```

2. **Edit the `.env` file and replace placeholders with your actual values:**

   - `GEMINI_API_KEY`: Your Google Gemini API key from [Google AI Studio](https://makersuite.google.com/app/apikey)
   - `QDRANT_API_KEY`: Your Qdrant API key (if using Qdrant Cloud) or generate one for local instance
   - `JWT_SECRET`: Generate a secure random string (use: `openssl rand -hex 32`)

3. **Verify `.env` is in `.gitignore`:**
   The `.env` file should already be listed in the root `.gitignore` file.

4. **Never share your `.env` file:**
   - Do not commit it to version control
   - Do not share it in screenshots
   - Do not upload it to any public location
   - Do not include it in bug reports

## Required Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `GEMINI_API_KEY` | Google Gemini API Key | `AIzaSy...` |
| `QDRANT_URL` | Qdrant database URL | `http://localhost:6333` |
| `QDRANT_API_KEY` | Qdrant API Key | `eyJhbGc...` |
| `JWT_SECRET` | Secret for JWT token generation | `7746e5bc8aa...` |
| `ENVIRONMENT` | Application environment | `development` |
| `DEBUG` | Enable debug mode | `true` |

## Generating Secure Secrets

### JWT Secret
```bash
# Linux/Mac
openssl rand -hex 32

# Python
python -c "import secrets; print(secrets.token_hex(32))"
```

## Troubleshooting

If you get an error about missing environment variables:
1. Make sure the `.env` file exists in the `backend/` directory
2. Verify all required variables are set
3. Restart your application after changing `.env`

## Additional Security Tips

- Rotate your API keys regularly
- Use different keys for development and production
- Never hardcode secrets in your code
- Use environment variables for all sensitive data
